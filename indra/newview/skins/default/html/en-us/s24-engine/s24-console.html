S24 Console - Viewer Operating System Interface

S24 Console is your command-line interface to the viewer operating system.
Access: Advanced menu → S24 Console (or enable via debug settings)
Home Common Settings S24 Console Troubleshooting

Overview

The S24 Console is a power-user command-line interface that provides direct control over viewer internals. Think of it as the viewer's equivalent to Windows PowerShell, Linux Terminal, or macOS Terminal.

⚠ Warning: These are experienced user commands. Some commands (like shutdown) are intentionally BRUTAL and bypass all safety prompts. Use with caution!

Command Categories

  • System Commands: Console control (cls, help, list, exit)
  • Console Utilities: General purpose tools (echo, calc, time, uptime, version, whois, motd, web)
  • Diagnostic Commands: Inspection and debugging (uuid, scan, lookat, find, tp, top)
  • Chat Commands: Direct chat output (say, shout)
  • Settings Commands: Debug setting manipulation (get, set)
  • Floater Commands: Direct window control (open)
  • System Control: Viewer shutdown (shutdown)

System Commands

cls / clear

Clear the console output buffer.

cls

help [command]

Display help information. Without arguments shows general help; with command name shows detailed help for that command.

help
help get
help shutdown

list

Display all available commands with brief descriptions.

list

exit / quit

Close the S24 Console floater (does not exit viewer).

exit

Console Utilities

echo <text>

Print text back to the console. Useful for testing, scripting, or creating visual separators.

echo Hello World!
echo ═══════════════════════
echo Testing console output...

calc <expression>

Calculate mathematical expressions with proper order of operations.

calc 2 + 3 * 4              # = 14
calc (2 + 3) * 4            # = 20
calc 100 / 3                # = 33.333333
calc 512 / 2                # = 256
calc -5 + 10                # = 5
calc 17 % 5                 # = 2 (modulo)
Operators: + - * / % with ( ) for grouping. Order of operations: parentheses → multiply/divide/modulo → add/subtract

time

Display current Second Life time and local time.

time

uptime

Show how long the viewer has been running (useful for stability monitoring).

uptime

version / ver

Display viewer version and build information. Useful for bug reports and verifying your build.

version
ver                  # Short alias
Info displayed: Channel name, full version string with build number, version components, build configuration, and platform details.

whois <name|uuid>

Look up avatar identity information: display name, username, and UUID.

whois Foo                    # Partial name search (nearby avatars)
whois FooResident            # Full username
whois 12345678-1234-...      # Look up by UUID
Identity Information:
  • Display Name: User's chosen display name (can be changed)
  • Username: Permanent SL account name (resident.name format)
  • UUID: Unique avatar identifier (never changes)
Data is retrieved instantly from the viewer's name cache - no server request needed!
Note: Name search only finds nearby avatars. UUID lookup works for any cached avatar.

Diagnostic Commands

uuid <uuid>

Identify what a UUID represents (avatar, object, texture, inventory item, group, etc.).

uuid 00000000-0000-0000-0000-000000000000
uuid 12345678-1234-1234-1234-123456789abc

scan [range]

Radar functionality - list nearby avatars with distance and UUID. Works at unlimited range using coarse location data.

scan              # List all avatars in scene
scan 50           # List avatars within 50 meters
scan 200          # List avatars within 200 meters
Tip: If names show as [UUID...], increase the People floater range slider to cache more names.

lookat <name|uuid>

Focus camera on an avatar or object. Supports partial name matching (case-insensitive).

lookat Foo Resident
lookat foo           # Partial match
lookat 12345678-1234-1234-1234-123456789abc
Draw Distance Limitation: If target is beyond your RenderFarClip setting, camera focus may not work. Use tp command instead for distant targets.

find <name|uuid> / find clear

Select and beacon an object or avatar with a persistent magenta GL beacon. Works at unlimited range!

find Foo Resident     # Beacon an avatar
find transparent      # Beacon hard-to-see objects
find clear            # Remove active beacon
How it works: The beacon tracks moving avatars using coarse location data and renders a visible magenta line from ground to target position. Great for finding people over 200m away!

tp <destination>

Teleport to locations. Highly reliable for distant targets beyond draw distance.

tp home                      # Teleport to home location
tp back                      # Teleport to previous location
tp Foo Resident              # Teleport to avatar by name
tp 12345678-1234-...         # Teleport to avatar/object by UUID

top [mode] [options]

Performance Analysis Tool - Display ranked objects by various performance metrics. Essential for parcel/region management, lag hunting, and identifying problem content.

Sort Modes

  • cost, render - Object render cost/weight (default)
  • physics, phys - Physics cost
  • streaming - Streaming cost
  • triangles, tris - Triangle/polygon count
  • script, scripts - Scripted objects (sorted by cost)

Scope Options

  • -p, --parcel - Limit to current parcel only
  • -r, --region - Limit to current region only

Ownership Filters

  • -s, --self - Show only objects you own
  • -g, --group - Show only group-owned objects
  • -o UUID, --owner UUID - Filter by specific owner UUID

Display Options

  • -n N, --limit N - Show top N results (default: 10, max: 50)

Examples

# Basic usage
top                          # Top 10 by render cost (all regions)
top scripts                  # Top 10 scripted objects
top physics                  # Top 10 by physics cost

# Parcel management
top scripts -p -s            # YOUR scripted objects in YOUR parcel
top cost -p --limit 20       # Top 20 highest-cost objects in parcel
top physics -p -g            # Group-owned objects by physics in parcel

# Region analysis
top scripts -r               # All scripted objects in region
top triangles -r -n 25       # Top 25 highest-poly objects in region

# Owner filtering
top cost -o aaaabbbb-1234-5678-90ab-cdef12345678
                            # All objects by specific owner
top scripts -p -o UUID       # Tenant's scripts in their parcel

# Combined filters
top physics -p -s -n 15      # Your top 15 physics objects in parcel
Workflow: Use top to identify problem objects by UUID, then use find <uuid> to beacon and locate them in-world.
Note: Objects must have cost data calculated by the server. Newly-rezzed or distant objects may show zero cost until data arrives.

Chat Commands

say <text>

Send message in normal chat (20m range).

say Hello everyone!

shout <text>

Send message in shout chat (100m range).

shout EVERYONE LISTEN!

Settings Commands

get <setting_name> / get *wildcard*

Retrieve debug setting values. Supports exact match or wildcard search.

get DebugShowFPS              # Exact match
get RenderFarClip             # Get draw distance
get *texture*                 # List all texture settings
get *render*quality*          # Wildcard search

set <setting_name> <value>

Modify debug setting values with automatic type detection (BOOL, S32, U32, F32, STRING).

set DebugShowFPS true         # Enable FPS counter
set RenderFarClip 512         # Set draw distance to 512m
set RenderVolumeLODFactor 4.5 # Floating-point values

Floater Commands

open list

List all registered floater windows with their internal names and descriptions.

open list

open <floater_name>

Open a specific floater window instantly (bypasses menu navigation).

open inventory
open preferences
open world_map
open beacons
open build
open profile         # Opens YOUR profile
Note: Some floaters (like profile and avatar_textures) automatically use your own data when opened without parameters.

Protected Floaters

Some floaters require specific context or IDs and cannot be opened directly. The console will provide helpful guidance when you attempt to use these:

  • Session-based: impanel, incoming_call, outgoing_call (require active calls/IMs)
  • ID-required: event, classified, experience_profile (require specific IDs from search/world)
  • Selection dialogs: group_picker, avatar_picker (callback context required)
  • Properties: item_properties, task_properties (require inventory/object selection)
  • Preview: preview_* floaters (require assets from inventory)
  • Upload: upload_* floaters (require file context)
  • Object-based: inspect, openobject (require world object selection)

open close <floater_name>

Close a specific floater window.

open close world_map
open close inventory

open toggle <floater_name>

Toggle a floater's visibility (show if hidden, hide if shown).

open toggle beacons
open toggle preferences
Common Floaters: inventory, preferences, world_map, camera, gestures, people, appearance, search, nearby_chat, beacons, build, snapshot, destinations, help_browser, map, scene_load_stats, debug_console, region_info, statistics, etc.

System Control Commands

shutdown now

BRUTAL immediate viewer exit. No prompts, no Ares cleanup, just STOP.

shutdown now
⚠ POWER USER COMMAND: This is NOT a graceful exit. The client will be GONE immediately. Use for testing, automation, or emergency exits only.

shutdown <seconds>

Schedule a shutdown after a countdown timer. Displays warnings at 10s, 5s, and 3s intervals.

shutdown 60              # Quit in 60 seconds
shutdown 300             # Quit in 5 minutes (AFK protection)

shutdown cancel

Abort an active shutdown timer.

shutdown cancel

# 1. Scan for nearby avatars

# 2. Identify an interesting UUID from the list

find 12345678-1234-1234-1234-123456789abc

# 4. Teleport to their location if needed

set RenderVolumeLODFactor 6.0

# Convert percentage to multiplier

set RenderAvatarMaxNonImpostors 1.25

Tips & Best Practices

  • Commands are case-insensitive: GET, get, Get all work
  • Line history: Use ↑/↓ arrow keys to recall previous commands
  • Partial name matching: lookat foo finds "Foo Resident"
  • Unlimited range: scan and find work beyond draw distance
  • Beacon persistence: Beacons track moving avatars automatically
  • Wildcard search: Use * in get commands to find related settings
  • Shutdown is BRUTAL: Experienced users only - it's an immediate STOP!

Color Coding

The console uses color coding to distinguish command feedback from system log messages:

Command Feedback (Your Actions):

  • ✓ Green: Success - operations completed successfully
  • ✗ Red: Error - operations failed or invalid input
  • ⚠ Yellow: Warning - important alerts from commands
  • Cyan: Info - helpful guidance and tips

System Log Messages (LL Output):

  • White: All LL warnings, errors, and info messages display as white text
  • Timestamps and [WARN]/[ERROR] tags are preserved for context
Why this approach? By coloring only your command feedback, you can instantly tell the difference between:
  • ✓ Opened floater: inventory (your command succeeded - green)
  • 2026-04-29 [INFO]: Inventory loaded (LL system message - white)
This makes it easy to scan the console and see what YOU did vs what the system reported.

Privacy & Safety Notes

  • Help examples use privacy-safe placeholders (not real UUIDs or names)
  • TPV Policy compliant - all commands respect LL's viewer boundaries
  • No remote execution - all commands are local to your viewer only
  • No automated scanning - all actions require explicit user commands
  • Easter eggs hidden - fun commands exist but aren't advertised in help
The S24 Console is an evolving system. New commands and features are added as the S24 viewer grows. Type list to see the current command set, or help <command> for detailed documentation.
